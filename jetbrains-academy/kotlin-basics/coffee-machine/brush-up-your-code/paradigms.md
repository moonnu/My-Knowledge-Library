# Paradigms

## Object-oriented programming

객체 지향 프로그래밍 또는 OOP 는 클래스의 집합으로 작성되는 패러다임 입니다. 각 클래스에는 개체라 불리는 인스턴스가 있습니다.

클래스(class)는 객체를 일반적으로 기술하는 방법으로, 개체와 상호 작용하기 위한 일반적인 규칙뿐만 아니라 해당 상태에 따라 달라지는 일반적인 상태와 동작을 정의합니다. 형식적으로 클래스는 필드, 속성, 클래스 및 함수와 같은 데이터 집합으로 표시됩니다.

예를 들어, 고양이라는 개체(entity)가 있고 이를 클래스를 사용하여 설명한다면, 고양이는 Cat 클래스의 객체(object)입니다. 고양이는 꼬리, 네 발, 발톱, 귀, 수염, 코와 같은 몇 가지 속성을 가지고 있습니다. 또한 고양이의 특징으로 야옹거리고, 먹고, 벽지를 찢는 등이 존재합니다. 이와 같은 모든 것들이 고양이의 메서드(method)가 될 수 있습니다.

OOP 는 현실에 존재하는 거의 모든 종류의 문제를 처리할 수 있습니다.

OOP 패러다임을 구현한 프로그래밍 언어는 Ruby, Java, C++, Python, Simula, VB, .NET 등이 있습니다.
